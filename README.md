# CaptureTranslater

Прототип overlay-переводчика для манги, маньхуа и любых страниц, где текст нужно читать прямо поверх другого приложения.

Текущая версия использует `PySide6/Qt`. Preview рисуется в `QOpenGLWidget`, если модуль `QtOpenGLWidgets`
доступен; иначе приложение использует QWidget fallback. Захват экрана выполняется в отдельном Qt-потоке через
`mss`, поэтому GUI не должен ждать каждый кадр.

## Возможности

- live preview экрана с выбором FPS до 30;
- красная рамка зоны OCR, которую можно двигать и растягивать мышкой;
- колесо мыши над preview приближает и отдаляет экран;
- зажатое колесико двигает трансляцию как карту;
- preview замазывает только свою область внутри захваченного кадра, чтобы не было бесконечного зеркала;
- обычные системные скриншоты снова видят окно приложения;
- пример окна перевода масштабируется как объект на реальном экране;
- настройки автоматически загружаются из `capture_translater.settings.json`;
- изменения применяются к рабочему overlay/OCR после кнопки `Сохранить`;
- transparent always-on-top overlay поверх других приложений;
- режим редактирования overlay: окна можно двигать, а двойной клик скрывает/возвращает окно;
- настройка обводки текста: цвет и толщина;
- добавление своих `.ttf`, `.otf` и `.ttc` шрифтов из интерфейса;
- кнопка `Сканировать область` захватывает сохраненную красную область и выводит найденные OCR-блоки в overlay;
- OCR и перевод выполняются в worker-потоке, поэтому окно настроек не должно зависать во время скана;
- перед OCR приложение кратко прячется и делает чистый снимок области, чтобы не распознавать собственное окно;
- переводчик объединяет OCR-блоки в крупные запросы, чтобы не ждать десятки сетевых вызовов подряд;
- OCR и перевод кешируются для повторного скана той же страницы;
- OCR-блоки фильтруются и объединяются, чтобы overlay не рассыпался на десятки мелких окон;
- выбор OCR-пресета: английский, японский, китайский/смешанный, сложная страница, Tesseract и будущие AI-режимы;
- icon-only шрифты вроде Segoe Fluent Icons автоматически заменяются читаемым шрифтом при рисовании текста;
- подробное логирование в `logs/capture_translater_YYYY-MM-DD_HH-MM-SS.log` с ротацией файлов.

## OCR

Встроен безопасный fallback OCR: если выбранный движок не подключен, приложение рисует тестовое окно. Это нужно,
чтобы проверять механику overlay, координаты и UX без тяжелых зависимостей.

Основной путь проекта - PaddleOCR. Для него в приложении уже есть пресеты:

- `PaddleOCR: английский ультрабыстрый`;
- `PaddleOCR: английский сбалансированный`;
- `PaddleOCR: английский точный`;
- `PaddleOCR: японский`;
- `PaddleOCR: китайский + английский`;
- `PaddleOCR: сложная страница`.

Установка PaddleOCR CPU-версии обычно делается двумя шагами:

```powershell
py -3.13 -m pip install paddlepaddle -i https://www.paddlepaddle.org.cn/packages/stable/cpu/
py -3.13 -m pip install "paddleocr[all]"
```

На Windows адаптер запускает PaddleOCR через `paddle_dynamic` CPU engine. Это обходит известное падение
static/oneDNN inference на некоторых связках PaddlePaddle 3.x.

Если в окружении есть `pytesseract` и `Pillow`, можно выбрать пресет `Tesseract: простой текст`:

```powershell
py -3.13 -m pip install pillow pytesseract
```

Сам бинарник Tesseract OCR тоже должен быть установлен в системе и доступен из `PATH`. Сейчас результат OCR
проходит через переводчик, если установлен `deep-translator`:

```powershell
py -3.13 -m pip install deep-translator
```

Если переводчик не установлен или не сработал, overlay покажет исходный OCR-текст и запишет причину в лог.

## Структура

- `models.py` - настройки, область перевода и стиль окна;
- `boxes.py` - модель переводного окна;
- `settings_store.py` - загрузка и сохранение JSON;
- `capture.py` - поток захвата экрана через `mss`;
- `preview.py` - OpenGL/Qt preview, zoom, pan и красная рамка;
- `overlay.py` - прозрачное topmost-окно поверх рабочего стола;
- `ocr.py` - интерфейс OCR и текущие движки;
- `ocr_presets.py` - список OCR-пресетов и их параметры;
- `translation.py` - перевод OCR-текста на русский;
- `pipeline.py` - связка `захват области -> OCR -> overlay boxes`;
- `scan_worker.py` - фоновый запуск медленного OCR/перевода без блокировки UI;
- `font_manager.py` - загрузка пользовательских шрифтов;
- `text_painter.py` - единое рисование текста с обводкой;
- `logging_config.py` - настройка файлового и консольного логирования;
- `main_window.py` - панель настроек и связывание компонентов;
- `app.py` - минимальная точка входа.

## Запуск

```powershell
py -3.13 -m pip install -r requirements.txt
py -3.13 main.py
```

## Ближайший слой

Рабочий цикл уже идет по схеме `захват красной области -> OCR -> перевод -> подбор окон -> показ поверх`.
Следующий слой - ускорить и очистить результат: фильтровать мелкий мусор OCR, объединять соседние строки,
кешировать повторные переводы и добавить AI-пресеты для сложных страниц.
